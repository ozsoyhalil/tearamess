CRITICAL pitfalls from RESEARCH.md:
- profiles join via follows FK: Use two-query pattern (fetch follow IDs, then query profiles separately with .in('user_id', ids)) — NOT PostgREST FK join syntax which may fail
- Feed: two-query merge (reviews + visits separately), not SQL UNION via Supabase client
- Cursor pagination: .lt('created_at', cursor) — NOT offset-based .range()
- visits upsert: use .upsert({...}, { onConflict: 'user_id,place_id' }) to avoid unique constraint errors
- recordVisit should also be called from createReview so visiting = writing a review (auto-visit on review, per Claude's Discretion)
