Create `src/components/WishlistButton.tsx`. Mirror the FollowButton pattern exactly.

Props: `{ placeId: string, initialIsWishlisted: boolean }`

State: `isWishlisted`, `isHovering`, `isPending`

Click handler:
1. If `!user` → `router.push('/auth/login')`, return
2. If `isPending` → return (debounce)
3. Capture `prev = isWishlisted`
4. `setIsPending(true)`, `setIsWishlisted(!isWishlisted)` (optimistic)
5. Call `addToWishlist(user.id, placeId)` or `removeFromWishlist(user.id, placeId)`
6. If `result.error` → revert: `setIsWishlisted(prev)`
7. `setIsPending(false)`

Visual states (per CONTEXT.md decisions):
- Not wishlisted: bookmark icon (outline) + "Gideceğim Yerler" label
- Wishlisted + not hovering: bookmark icon (filled, caramel color) + "Listede"
- Wishlisted + hovering: bookmark icon (filled, warmgray) + "Listeden Çıkar"
- Pending: opacity-50, cursor-not-allowed

Use SVG bookmark icon inline (no icon library needed). Caramel color: `text-caramel-600` or `text-[#C08552]`.

Tailwind classes follow the Tiramisu token pattern from the rest of the app. Use `onMouseEnter`/`onMouseLeave` for hover state (NOT CSS :hover pseudo-class — needs state-driven label swap).

Named export: `export function WishlistButton`.
