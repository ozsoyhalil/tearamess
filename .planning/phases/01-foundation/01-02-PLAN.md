  <action>
    **Step 1: Create `src/types/place.ts`**

    Merge the inline Place types from explore and place detail pages into one canonical type that covers all usages:

    ```typescript
    export interface Place {
      id: string
      name: string
      slug: string
      category: string
      city: string
      neighborhood: string | null
      description?: string | null
      created_by?: string
      created_at?: string
      avg_rating?: number | null
      review_count?: number
    }
    ```

    **Step 2: Create `src/types/profile.ts`**

    ```typescript
    export interface Profile {
      id?: string
      user_id?: string
      username: string | null
      display_name: string | null
      avatar_url?: string | null
      created_at?: string
    }
    ```

    **Step 3: Create `src/lib/services/places.ts`**

    All functions import `supabase` from `@/lib/supabase` (NOT from `@/lib/supabase-server`). Return shape is always `{ data: T | null, error: string | null }`.

    Functions to implement:
    - `getPlaces(category?: string, city?: string)` — fetches all places with optional filters, computes `avg_rating` and `review_count` from joined reviews. SELECT: `id, name, slug, category, city, neighborhood, reviews(rating)`. Compute avg/count from reviews array.
    - `searchPlaces(query: string)` — ilike search on name, limit 8. SELECT: `id, name, slug, category, city`.
    - `getPlaceBySlug(slug: string)` — single place by slug. SELECT: `*`. Returns `{ data: Place | null, error: string | null }`.
    - `createPlace(payload: { name: string, slug: string, category: string, city: string, neighborhood?: string, description?: string, created_by: string })` — inserts and returns `{ data: null, error: string | null }`.

    **Step 4: Create `src/lib/services/reviews.ts`**

    - `getReviewsForPlace(placeId: string)` — SELECT `*, profiles(username, display_name)` WHERE place_id = placeId, ORDER BY created_at DESC. Returns `Review[]`.
    - `createReview(payload: { place_id: string, user_id: string, rating: number, content: string | null, visit_date: string | null })` — insert into reviews. Returns `{ data: null, error: string | null }`.
    - `getUserReviews(userId: string)` — SELECT `*, places(name, slug, category, city)` WHERE user_id = userId, ORDER BY created_at DESC. Returns `{ data: any[] | null, error: string | null }`.

    **Step 5: Create `src/lib/services/profiles.ts`**

    - `getProfileByUserId(userId: string)` — SELECT `username, display_name, avatar_url` FROM profiles WHERE user_id = userId, single. Returns `{ data: Profile | null, error: string | null }`.
    - `getProfile(userId: string)` — same as above, alias kept for callers.

    **Step 6: Create `src/lib/services/auth.ts`**

    Wraps Supabase auth methods. Import `supabase` from `@/lib/supabase`.
    - `signIn(email: string, password: string)` — calls `supabase.auth.signInWithPassword`. Returns `{ data: null, error: string | null }`.
    - `signUp(email: string, password: string, metadata: { username: string, display_name: string })` — calls `supabase.auth.signUp`. Returns `{ data: null, error: string | null }`.
    - `signOut()` — calls `supabase.auth.signOut`. Returns `{ error: string | null }`.
    - `getSession()` — calls `supabase.auth.getSession`. Returns the session or null.

    **Step 7: Fill in test stubs** in `src/lib/services/places.test.ts` and `src/lib/services/reviews.test.ts` — replace the `it.todo` stubs with real assertions using jest.fn() mocks for the supabase client. At minimum make the happy-path and error-path tests pass for `getPlaces` and `getReviewsForPlace`.
